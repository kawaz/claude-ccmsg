# DR-0022: In-view キーワード検索 (Timeline / ファイルビューアのハイライト検索)

Status: Accepted (kawaz r26 mid=59 が仕様を直接指定、裁定項目なし)
Date: 2026-07-17
Sponsor: kawaz r26 mid=59

## 1. 背景

表示中の Timeline / ファイルビューア内をキーワード検索したい (kawaz)。ブラウザ標準の
Cmd+F は standalone PWA では使えず、fold された TL 構造にも効かない。

## 2. 仕様 (kawaz mid=59 の指定を転記)

### 2.1 検索 UI

- どこかに 🔍 ボタン → 押すとクエリ入力が出る
- **[Aa] / [.*] の 2 トグルボタン**: それぞれ case sensitive / regex
- クエリはマルチライン可。通常検索は**行内の空白区切りワードを OR、改行区切りを AND** とする。ダブルクオートで囲んだ句は 1 ワードとして扱い、句内の連続空白は `\s+` マッチに正規化する
- regex 検索は行内を分割せず、各行を独立した正規表現として改行区切り AND にする。空行だけを無視し、各行の空白は trim しない
- 入力欄外クリックで入力欄を閉じ、**各ワードを別チップで一列表示**
- 表示イメージ:
  - 検索なし: `🔍`
  - 検索あり: `🔍 foo bar [1/20]↑↓`
- TL / コードでは**AND 行毎に別色**でハイライトし、同じ行の OR ワードは同色にする

### 2.2 スクロール検索インデックス (ユーザメッセージ 👤 nav も同仕様に統一)

- `[1/20]` = 現在インデックス / 総マッチ数。スクロール動作は既存のユーザメッセージ移動と同様
- **ハイライトをビューア上でクリック** → `1/20` の現在値がその要素のインデックスに更新
  (この数字変化ではスクロールしない。スクロールを伴う数字変化は ↑↓ ボタンのみ)
- ↑↓ のインデックスは **1 と最大値を越えたらループ**

## 3. 設計方針

- webui 完結 (daemon 変更なし)。表示済みコンテンツ (TL の読み込み済み範囲 / ビューアの
  表示ファイル) に対するクライアントサイド検索
- クエリから直列化可能な AND/OR 正規表現構造を作る parser は protocol package の pure 層に置き、in-view 検索と Session Search で共有する。不正 regex は構造内のエラーとして返す
- in-view 固有のマッチ列挙・AND/OR 判定は webui の pure 層に切り出してテスト
- ハイライトは既存 render 済み DOM への後段 mark ではなく、render 時に text segment を
  分割して <mark> を差し込む方向 (preact 流)。ワード色は palette 固定循環
- 👤 nav (ユーザメッセージ移動) のインデックス管理を § 2.2 仕様に合わせて共通化

## 4. Phase 分割

| Phase | スコープ |
|---|---|
| Phase 1 | pure 層 (クエリ parse / マッチ列挙 / index 遷移) + ファイルビューア検索 |
| Phase 2 | Timeline 検索 (fold との相互作用込み) + 👤 nav の仕様統一 |
