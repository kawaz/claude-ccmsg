# kawaz 発言: webui の最終形 vision とフロントエンド構成の好み (2026-07-09)

一次資料。webui phase (DR-0004 実装直後) のセッションで kawaz が表明した長期 vision と構成の好み。DR-0005 の根拠。

出典セッション: claude-ccmsg セッション `5921d764-e8c1-4dad-9cf8-f9453ef5bc85`

> ちなみにwebuiだが、agentとのメッセージングは勿論だが、最終的にはセッションのjsonlのリッチビューアとしても機能するようになると嬉しい。
>
> イメージとしてはccmsg付きのセッション一覧、ccmsgのルーム一覧、セッションを選ぶとjsonlからセッションをリッチにレンダリング、プロジェクト内のファイルツリーとその中のファイル選択したらファイルビューアも欲しい。など最終的にそこで基本的な作業が全てできるものに育つととても良い。
> 一気に最終形は目指せないだろうけど、そこまでも見込んだ設計になってると素敵。なので雑にペラ1の詰め込みHTML＋JSトかにするんじゃなく、コンポーネント化や各種アクションやメッセージが形式化してリデューサーで処理するような構成になってると好み。

注: 「最終形を一気に目指す」要求ではなく「そこまで見込んだ設計 (器)」の要求。機能要求 (セッション一覧 / セッション jsonl リッチレンダリング / ファイルツリー + ファイルビューア) は roadmap、構成要求 (コンポーネント化 + 形式化された action/メッセージ + reducer) は即時のアーキテクチャ制約。
